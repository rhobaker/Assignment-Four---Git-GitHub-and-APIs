The README File

What are README Files for?

README.md files are used to explain your project to other people, as it is the first thing they will see when looking at your project in GitHub.

It will explain what your project is for and how other people can interact with it.

What should be in a README file?

The following items should be included within the file:

  • A title
  • A description of the project
  • How to use or interact with the project
  • Any programs used in the project
  • Licence details
  • Collaborators

README formatting

Markdown is used to format README files.

Headers are especially important, as they will generate a table of contents in GitHub.

Images and videos can also be incorporated if needed.
