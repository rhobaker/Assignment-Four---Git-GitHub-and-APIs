The .gitignore file

The .gitignore file contains all of the files and folders that you want Git to ignore in the repository.

These will then not be listed and will not be available to stage or commit.

How to create the file

The file is created in plain text and has to be called .gitignore.txt.

It contains a list of all the files and folders that you want Git to ignore and not track in the repository.

Each additional file that you want to ignore should be on a new line within the file.

The lines in the file can also use symbols to create matching patterns.

e.g. * is a wildcard match.

# can be used to add comments to explain the lines in the file.


Example lines:

# Ignore Mac system files

.DS_store


# Ignore all text files

*.txt


How to add .gitignore.txt

The text file should be saved into the root directory of the Git folder and be called .gitignore.txt.

To create the file, navigate to the root directory in the terminal and use the command:

touch .gitignore

You can then open the file (remember it will be hidden) and add a list of the files you want to ignore.
